# opencv-object-counting 开发笔记

## 项目目标
基于传统计算机视觉方法，实现对规则目标的分割、轮廓提取、目标计数与结果标注。

当前计划以 **药片/胶囊图片** 为主要对象，逐步完成从简单样例图到复杂样例图的检测流程。

---

## 第一阶段目标：完成基础检测流程

### 阶段任务
1. 完成项目基础目录和工程配置
2. 读取输入图片并正常显示
3. 对简单药片图像完成预处理
4. 实现目标分割
5. 提取轮廓并完成初步计数
6. 将检测结果用框或轮廓绘制出来

---

## 第一阶段输入数据
当前优先使用简单样例图：
- 背景单一
- 目标边界清晰
- 大部分目标彼此分离
- 目标数量适中

建议首张测试图：
- `assets/input/pill1.jpg`

---

## 第一阶段技术路线
### 1. 图像读取
- 使用 OpenCV 读取图片
- 检查图像是否成功加载
- 显示原图用于调试

### 2. 图像预处理
- 灰度化
- 高斯滤波去噪
- 根据效果尝试二值化方法：
  - 固定阈值
  - Otsu 阈值
  - 自适应阈值（后续可选）

### 3. 目标分割
- 将药片从背景中分离出来
- 观察是否存在阴影、边缘断裂、背景干扰等问题
- 必要时使用形态学操作优化结果：
  - 开运算去噪
  - 闭运算填补小空洞

### 4. 轮廓提取
- 使用 `findContours` 提取外轮廓
- 绘制轮廓观察检测效果
- 输出轮廓数量作为初步计数结果

### 5. 目标筛选
- 根据面积过滤噪声轮廓
- 视情况增加长宽比、轮廓尺寸等筛选规则
- 减少误检

### 6. 结果可视化
- 在原图上绘制：
  - 外接矩形
  - 轮廓
  - 编号或总数
- 输出最终计数图

## 第一阶段预期成果
完成后应至少实现以下功能：
- 输入一张简单药片图
- 正确完成目标分割
- 提取主要轮廓
- 输出目标数量
- 在原图上标注检测结果

---

## 第一阶段可能遇到的问题
1. 药片阴影影响二值化结果
2. 白色药片与浅色高光区域对比不足
3. 轮廓边缘可能不够完整
4. 小噪声区域可能被误识别为目标
5. 少量接近目标可能被当成一个整体轮廓

---

## 第一阶段完成标准
满足以下条件即可认为第一阶段完成：
- 程序可稳定读取并处理简单样例图
- 对简单药片图的计数结果基本正确
- 检测结果可视化清晰
- 代码结构不全部堆在 `main` 中
